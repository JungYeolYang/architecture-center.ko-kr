## <a name="solution"></a><span data-ttu-id="a9773-119">해결 방법</span><span class="sxs-lookup"><span data-stu-id="a9773-119">Solution</span></span>

<span data-ttu-id="a9773-120">메시지 큐를 사용해 응용 프로그램과 소비자 서비스의 인스턴스 사이에 통신 채널을 구현합니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-120">Use a message queue to implement the communication channel between the application and the instances of the consumer service.</span></span> <span data-ttu-id="a9773-121">응용 프로그램은 메시지 형태의 요청을 큐에 게시하고, 소비자 서비스 인스턴스는 메시지를 큐에서 수신해 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-121">The application posts requests in the form of messages to the queue, and the consumer service instances receive messages from the queue and process them.</span></span> <span data-ttu-id="a9773-122">이 방법을 통해 소비자 서비스 인스턴스의 동일한 풀을 사용하여 응용 프로그램의 인스턴스에서 전송한 메시지를 처리할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-122">This approach enables the same pool of consumer service instances to handle messages from any instance of the application.</span></span> <span data-ttu-id="a9773-123">다음 그림은 메시지 큐를 사용하여 작업을 서비스의 인스턴스로 분산시키는 과정을 보여 줍니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-123">The figure illustrates using a message queue to distribute work to instances of a service.</span></span>

<span data-ttu-id="a9773-125">이 솔루션에는 다음과 같은 이점이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-125">This solution has the following benefits:</span></span>

- <span data-ttu-id="a9773-136">확장이 가능합니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-136">It's scalable.</span></span> <span data-ttu-id="a9773-137">시스템은 메시지의 양이 변함에 따라 소비자 서비스의 인스턴스 수를 동적으로 늘리거나 줄일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a9773-137">The system can dynamically increase or decrease the number of instances of the consumer service as the volume of messages fluctuates.</span></span>
